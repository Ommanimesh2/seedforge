# v2.2.0 — LLM-Enhanced Text Generation

**Codename:** `muse`
**Breaking:** No
**Estimated scope:** ~1,000 LOC new

---

## Motivation

Faker produces grammatically correct but semantically empty text. For columns like `product_description`, `review_body`, `bio`, or `article_content`, LLM-generated text is dramatically more realistic. Snaplet/Seed already offers this via OpenAI. Our differentiator: **local model support (Ollama) with zero vendor lock-in**.

---

## Design Principles

1. **Opt-in** — never call an LLM unless explicitly configured
2. **Offline-first** — Faker is always the fallback, LLM is an enhancement
3. **Provider-agnostic** — OpenAI, Anthropic, Groq, Ollama (local) via a unified interface
4. **Batched** — generate all LLM text in one pass before row insertion to minimize API calls
5. **Cached** — cache LLM responses by (table, column, seed) to avoid re-generation
6. **Cost-aware** — show estimated token usage before generation, require confirmation

## Feature Design

### CLI

```bash
# Use default provider (auto-detects OPENAI_API_KEY, ANTHROPIC_API_KEY, or local Ollama)
seedforge --db postgres://localhost/mydb --ai

# Specify provider
seedforge --db postgres://localhost/mydb --ai --ai-provider ollama
seedforge --db postgres://localhost/mydb --ai --ai-provider openai --ai-model gpt-4o-mini

# Only enhance specific columns
seedforge --db postgres://localhost/mydb --ai --ai-columns products.description,reviews.body
```

### Config

```yaml
# .seedforge.yml
ai:
  enabled: true
  provider: ollama          # openai | anthropic | groq | ollama
  model: llama3.2           # provider-specific model name
  baseUrl: http://localhost:11434  # for ollama or custom endpoints
  # apiKey: ${OPENAI_API_KEY}     # env var interpolation (existing feature)
  columns:
    - products.description
    - reviews.body
    - users.bio
    - posts.content
  temperature: 0.7
  maxTokensPerField: 200    # cap output length
  batchSize: 20             # generate 20 values per API call
  cache: true               # cache responses (default: true)
```

### Programmatic API

```typescript
import { seed } from '@otg-dev/seedforge'

await seed('postgres://localhost/testdb', {
  ai: {
    provider: 'ollama',
    model: 'llama3.2',
    columns: ['products.description', 'reviews.body'],
  },
})

// Or per-column in seeder
const seeder = await createSeeder(dbUrl)
await seeder.seed('products', 50, {
  columns: {
    description: {
      ai: true,  // use default AI provider
      prompt: 'Write a short e-commerce product description for a {category} product named {name}',
    },
  },
})
```

## Implementation

### Phase 1: Provider interface + Ollama (~300 LOC)

1. Define `AIProvider` interface:
   ```typescript
   interface AIProvider {
     generate(requests: AIGenerationRequest[]): Promise<string[]>
     estimateTokens(requests: AIGenerationRequest[]): number
     name: string
   }

   interface AIGenerationRequest {
     table: string
     column: string
     columnType: string       // e.g., "product description", "user bio"
     context: Record<string, unknown>  // other column values in the same row
     maxTokens: number
   }
   ```

2. Implement `OllamaProvider` first (no API key needed, best for testing):
   - HTTP POST to `http://localhost:11434/api/generate`
   - Batch via multi-turn or repeated calls
   - Detect if Ollama is running, helpful error if not

3. Tests: Mock HTTP server for Ollama responses.

### Phase 2: OpenAI + Anthropic providers (~300 LOC)

1. `OpenAIProvider`:
   - Uses `fetch()` to `https://api.openai.com/v1/chat/completions`
   - Supports any OpenAI-compatible endpoint (Groq, Together, etc.) via `baseUrl`
   - Batch: send multiple values in one prompt

2. `AnthropicProvider`:
   - Uses `fetch()` to `https://api.anthropic.com/v1/messages`
   - Same batch strategy

3. **Zero new dependencies** — use native `fetch()` (Node 18+), no SDK packages.

4. Provider auto-detection:
   - `OPENAI_API_KEY` set → OpenAI
   - `ANTHROPIC_API_KEY` set → Anthropic
   - `GROQ_API_KEY` set → OpenAI-compatible with Groq base URL
   - Ollama running on localhost → Ollama
   - None → error with helpful message

### Phase 3: Batch orchestrator + prompts (~250 LOC)

1. `src/ai/batch.ts`:
   - Collects all (table, column) pairs marked for AI generation
   - Groups by column domain (all product descriptions together, all bios together)
   - Generates in batches of N (configurable, default 20)
   - Returns `Map<string, string[]>` — key is `table.column`, value is generated texts

2. `src/ai/prompt.ts`:
   - Domain-aware prompt templates:
     ```
     "Generate 20 unique product descriptions for an e-commerce store.
      Each should be 1-2 sentences. Return as a JSON array of strings."
     ```
   - Context injection: if other columns in the row are known (e.g., product name, category), include them in the prompt
   - Template variables: `{name}`, `{category}`, `{table}`, `{column}`

3. Integration with generator:
   - Before row generation, run AI batch for all AI-marked columns
   - Store results in a pool
   - During row generation, pull from the AI pool instead of Faker

### Phase 4: Cache + cost estimation (~150 LOC)

1. `src/ai/cache.ts`:
   - File-based cache at `.seedforge-cache/ai/`
   - Key: hash of (table, column, seed, count, model)
   - Value: JSON array of generated strings
   - TTL: configurable (default: 7 days)
   - `--no-cache` flag to bypass

2. `src/ai/cost.ts`:
   - Estimate tokens based on prompt template + expected output
   - Display before generation: "Estimated: ~5,000 tokens ($0.002 with gpt-4o-mini)"
   - Require `--yes` or interactive confirmation for paid providers

## Success Criteria

- [ ] `--ai` flag with Ollama generates realistic text for TEXT/VARCHAR columns
- [ ] OpenAI and Anthropic providers work with API keys
- [ ] Auto-detection picks the right provider from environment
- [ ] Batch generation minimizes API calls (20+ values per call)
- [ ] Cache prevents re-generation on repeated runs with same seed
- [ ] Cost estimation shown before paid API calls
- [ ] Column-level opt-in: only specified columns use AI
- [ ] Context injection: AI sees other row values for coherent output
- [ ] Graceful fallback to Faker if AI provider unavailable
- [ ] Zero new npm dependencies (native fetch)
- [ ] ≥30 new tests (mocked providers)
- [ ] Works with programmatic API

---

## Non-goals

- Image generation for BYTEA/BLOB columns
- AI-generated JSON structures
- Fine-tuned models or custom training
- Embedding generation
