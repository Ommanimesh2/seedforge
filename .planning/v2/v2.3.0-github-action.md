# v2.3.0 — GitHub Action + CI/CD Integration

**Codename:** `pipeline`
**Breaking:** No
**Estimated scope:** ~600 LOC new (action + workflow templates)

---

## Motivation

Seeding a test database is one of the most common CI steps for integration tests. Today users must manually write `npx seedforge --db $DATABASE_URL` in their workflows. A first-class GitHub Action with caching, reporting, and idempotent runs makes seedforge the obvious choice for CI pipelines.

---

## Feature Design

### GitHub Action

```yaml
# .github/workflows/test.yml
jobs:
  test:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_DB: testdb
          POSTGRES_PASSWORD: postgres
        ports: ['5432:5432']
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

    steps:
      - uses: actions/checkout@v4

      - uses: seedforge/action@v1
        with:
          db: postgres://postgres:postgres@localhost:5432/testdb
          count: 50
          seed: 42
          config: .seedforge.yml       # optional
          exclude: schema_migrations   # optional
          node-version: '20'           # optional (default: 20)

      - run: npm test
```

### Action features

1. **Auto-setup** — installs seedforge, runs seeding
2. **Schema caching** — caches introspected schema hash to skip re-introspection when schema hasn't changed
3. **Idempotent** — detects if tables already have data, skips or cleans based on config
4. **PR comment** — posts seed summary as a PR comment (opt-in)
5. **Output variables** — exposes `tables`, `rows`, `duration` as step outputs
6. **Multi-DB** — supports PG, MySQL, SQLite
7. **Schema-only** — supports `--prisma`, `--drizzle` flags for schema-file-based seeding

### Action outputs

```yaml
- uses: seedforge/action@v1
  id: seed
  with:
    db: ${{ env.DATABASE_URL }}

- run: echo "Seeded ${{ steps.seed.outputs.rows }} rows across ${{ steps.seed.outputs.tables }} tables in ${{ steps.seed.outputs.duration }}ms"
```

## Architecture

### New repository: `seedforge/action`

```
seedforge-action/
├── action.yml              # GitHub Action metadata
├── src/
│   ├── main.ts             # Action entry point
│   ├── inputs.ts           # Parse action inputs
│   ├── cache.ts            # Schema hash caching
│   ├── comment.ts          # PR comment posting
│   └── outputs.ts          # Set step outputs
├── dist/
│   └── index.js            # Bundled output (ncc compiled)
├── package.json
├── tsconfig.json
└── README.md
```

### Dependencies

- `@actions/core` — inputs, outputs, logging
- `@actions/cache` — schema hash caching
- `@actions/github` — PR comment posting
- `@otg-dev/seedforge` — the actual seeding (programmatic API from v2.0.0)

## Implementation

### Phase 1: Basic action (~250 LOC)

1. `action.yml`:
   ```yaml
   name: 'seedforge'
   description: 'Seed your test database with realistic data'
   inputs:
     db:
       description: 'Database connection string'
       required: false
     prisma:
       description: 'Path to Prisma schema file'
       required: false
     drizzle:
       description: 'Path to Drizzle schema file'
       required: false
     count:
       description: 'Rows per table'
       default: '50'
     seed:
       description: 'PRNG seed for deterministic output'
     config:
       description: 'Path to .seedforge.yml'
     exclude:
       description: 'Comma-separated tables to skip'
     output:
       description: 'Path to export SQL file'
     node-version:
       description: 'Node.js version to use'
       default: '20'
   outputs:
     tables:
       description: 'Number of tables seeded'
     rows:
       description: 'Total rows inserted'
     duration:
       description: 'Duration in milliseconds'
   runs:
     using: 'node20'
     main: 'dist/index.js'
   ```

2. `src/main.ts`:
   - Read inputs via `@actions/core`
   - Call seedforge programmatic API (`seed()` from v2.0.0)
   - Set outputs
   - Log summary using `core.summary`

### Phase 2: Schema caching (~150 LOC)

1. After introspection, hash the schema (table names + column types + FK structure).
2. Cache the hash using `@actions/cache` with key `seedforge-schema-{hash}`.
3. On next run, compare hash — if unchanged and data exists, skip seeding.
4. Cache key includes: schema hash + seedforge version + count + seed value.

### Phase 3: PR comment + polish (~200 LOC)

1. Opt-in PR comment:
   ```yaml
   - uses: seedforge/action@v1
     with:
       db: ${{ env.DATABASE_URL }}
       comment: true
   ```

2. Comment format:
   ```markdown
   ### 🌱 seedforge seed summary

   | Table | Rows | Duration |
   |-------|------|----------|
   | users | 50 | 12ms |
   | posts | 50 | 18ms |
   | comments | 50 | 15ms |

   **Total:** 150 rows across 3 tables in 45ms
   Seed: `42` | Config: `.seedforge.yml`
   ```

3. Update existing comment on re-run (don't spam).

## Success Criteria

- [ ] `uses: seedforge/action@v1` works with PG, MySQL, SQLite
- [ ] Schema-file inputs (prisma, drizzle) work without live DB
- [ ] Outputs (tables, rows, duration) are set correctly
- [ ] Schema caching skips unnecessary re-seeding
- [ ] PR comment posts cleanly with table summary
- [ ] Action runs in <30s for typical schemas (excluding DB setup)
- [ ] Works with GitHub-hosted and self-hosted runners
- [ ] README with copy-paste workflow examples
- [ ] Published to GitHub Marketplace

## Non-goals

- GitLab CI / CircleCI / Jenkins templates — future work
- Docker-based action (Node is faster to start)
- Database provisioning (user provides the service container)
